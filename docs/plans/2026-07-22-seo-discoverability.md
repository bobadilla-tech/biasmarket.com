# SEO and discoverability foundation for apps/web

## Context

No sitemap, robots.txt, structured metadata, OG image, or JSON-LD existed
anywhere in the repo. Bias Market is a store-builder — creator storefronts
(`/store/[slug]`) are the actual content that needs to rank and share well, not
just the marketing pages. Went from zero to a working baseline: crawl
directives, a generated sitemap covering both locales, page-level metadata,
a generated OG image, and per-store structured data.

## Decisions

- **Public stores need a list endpoint.** `Store` has no `updatedAt` or
  published flag, and the only public read was `GET /stores/:slug/public`
  (single store). Added `GET /stores/public` (slug + createdAt for every
  store) so `sitemap.ts` can enumerate storefronts without a new field on the
  model — matches the existing `findPublicBySlug` visibility (any store is
  public once created, no gating), and keeps web's data access through the api
  over HTTP per the multi-tenant hard rule.
- **`lastModified` on store sitemap entries uses `createdAt`, not real
  freshness.** No `updatedAt` column exists yet. Honest-but-imperfect signal is
  better than fabricating one; static pages get no `lastModified` at all for
  the same reason.
- **robots.txt disallows via path pattern, not per-page `noindex` meta.**
  Dashboard/onboarding/login/cart/checkout are all auth-gated or transactional
  — blocking crawl entirely (`disallow`) is the right mechanism; adding
  `noindex` meta on top would be redundant since a disallowed page never gets
  fetched for its meta tag to be read.
- **Sitemap hreflang alternates, not per-page `<link rel="alternate">` tags.**
  Getting real pathname into `generateMetadata` for global hreflang links would
  need request-header plumbing Next doesn't expose by default. Sitemap-level
  `alternates.languages` (one per locale, per URL) covers the same signal for
  Google with far less complexity — good enough for a first pass.
- **OG image generated by a script, not the `opengraph-image.tsx` file
  convention.** User asked for a script specifically. Used `next/og`'s
  `ImageResponse` (Satori + resvg, already bundled inside `next` — zero new
  deps) from a plain Node script rather than a dynamic per-request route, since
  the image is a static brand card, not data-dependent.
- **JSON-LD as one `@graph` block per store page**, not separate script tags.
  `OnlineStore` + one `Product`/`Offer` per listed product, output via
  `JSON.stringify(...).replace(/</g, "\\u003c")` — the XSS-safe pattern from
  Next's own JSON-LD guide (`node_modules/next/dist/docs/01-app/02-guides/json-ld.md`).

## What changed

**API** (`apps/api/src/modules/stores/`)

- `stores.service.ts` — `findAllPublic()`: `{slug, createdAt}` for every store.
- `stores.controller.ts` — `@Public() GET /stores/public`, backing the sitemap.

**Web**

- `apps/web/app/robots.ts` — allow `/`, disallow `/*/login`,
  `/*/onboarding*`, `/*/dashboard*`, `/*/store/*/cart`, `/*/store/*/checkout`
  (all locale-prefixed); points at `/sitemap.xml`.
- `apps/web/app/sitemap.ts` — static marketing pages (home, founder,
  enterprise) × both locales, plus every store's `/store/[slug]` × both
  locales (from the new API endpoint), each with `hreflang` alternates.
- `apps/web/app/[locale]/layout.tsx` — `metadataBase`, title template
  (`%s — Bias Market`), default OpenGraph/Twitter card metadata, default OG
  image.
- `apps/web/app/[locale]/(storefront)/store/[slug]/page.tsx` —
  `generateMetadata` (store name/description/OG image, `noindex` when the
  store 404s) + JSON-LD `<script>` block.
- `apps/web/lib/site-config.ts` — added `SITE_URL` (`NEXT_PUBLIC_SITE_URL` env
  var, falls back to `https://biasmarket.com`).
- `apps/web/scripts/generate-og-image.ts` — generates
  `apps/web/public/og-image.png` (1200×630) via `next/og`'s `ImageResponse`.
  Run with `pnpm --filter web og:generate`.

**Not done yet** — flagged as follow-up, not started: no `NEXT_PUBLIC_SITE_URL`
set in any env file yet (falls back to the hardcoded default); no canonical
`<link>` tags beyond what `metadataBase` implies; homepage has no
`Organization` JSON-LD (only store pages do).
